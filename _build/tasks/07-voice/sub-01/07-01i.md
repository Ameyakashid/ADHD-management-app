## Key Decisions

1. **kokoro-onnx over kokoro (PyTorch)**: No system deps, lighter install, simpler API. Meets "no manual steps" requirement.
2. **stdlib `wave` over `soundfile`**: Avoids extra dependency. WAV writing is ~6 lines with stdlib.
3. **Removed `_detect_gpu_available()`**: kokoro-onnx handles ONNX Runtime provider selection internally. No need for our wrapper to do GPU detection.
4. **Close fd before urlretrieve**: Windows file locking requires closing the `mkstemp` fd before another process writes to the temp file.
5. **All `synthesize_speech` parameters explicit**: No defaults, per code rules. Callers use exported `DEFAULT_*` constants.
