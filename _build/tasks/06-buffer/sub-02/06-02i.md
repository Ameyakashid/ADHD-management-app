## Files Created

### buffer_tools.py (299 lines)
Five nanobot-ai Tool subclasses wrapping BufferStore, plus two formatting helpers and a registration function.

**Formatting helpers:**
- `format_buffer(buffer)` — Single buffer → LLM-readable string with id prefix, level/capacity, status, next due date, days until due, recurrence interval, alert threshold. Includes "⚠ Below alert threshold" warning when level ≤ threshold.
- `format_buffer_list(buffers)` — List of buffers → joined format or "No buffers found."

**Tool classes:**
- `CreateBufferTool` — Wraps `store.create_buffer()`. Defaults `buffer_level` to `capacity` (full buffer) and `alert_threshold` to 1 when omitted. Parses `next_due_date` as `date.fromisoformat()` (not datetime). Catches ValueError for invalid dates and level > capacity.
- `ListBuffersTool` — Read-only. Defaults to `list_active_buffers()`. Optional `status` filter falls back to `list_buffers()` + Python filter.
- `GetBufferStatusTool` — Read-only. Wraps `store.get_buffer()`. Returns formatted single buffer.
- `RefillBufferTool` — Wraps `store.refill()`. Catches both KeyError and ValueError.
- `ManualDecrementTool` — Wraps `store.decrement()`. Catches both KeyError and ValueError.

**Registration:**
- `register_buffer_tools(registry, store)` — Registers all 5 tools.

## Key Decisions

1. **buffer_level defaults to capacity.** Research recommended Option B. A new buffer starts full — "I have 4 weeks banked." The LLM can override with an explicit `buffer_level` param.

2. **alert_threshold defaults to 1.** When omitted, threshold is 1. Simpler than capacity-based formula, and 1 is a reasonable "last unit" warning.

3. **date.fromisoformat() for next_due_date.** Research flagged that task_tools' `parse_iso_date()` returns `datetime`, not `date`. Used `date.fromisoformat()` directly instead of creating a new helper.

4. **ListBuffersTool defaults to active buffers.** Without a status filter, calls `list_active_buffers()`. With a status filter, fetches all and filters in Python. Matches the task_tools pattern of optional status filtering.

5. **Test file split.** Combined test file was 355 lines. Split format helper tests into `test_buffer_format.py` (64 lines) to keep main test file at 300 lines.

6. **No visual level indicator.** Research suggested `[|||..]` but noted it might be unnecessary. Skipped — the LLM can read `3/4` just fine.
