## Files Created

### buffer_hook.py (156 lines)
Buffer heartbeat hook for nanobot-ai. Contains:

**Pure functions (5):**
- `find_due_buffers(buffers, current_date)` — filters buffers where `next_due_date <= current_date`
- `format_buffer_alert_line(buffer)` — formats one buffer as factual data line (name, level/capacity, due date, interval)
- `format_buffer_alerts(alertable_buffers)` — wraps alert lines under `## Buffer Alerts` header; returns empty string if none
- `collect_alertable_buffers(buffers)` — filters buffers where `buffer_level <= alert_threshold`
- `inject_alerts_into_prompt(system_content, alert_block)` — appends alert block to system prompt

**BufferHook class:**
- Constructor takes `buffer_store: BufferStore`, `is_scheduled_session: Callable[[], bool]`, `get_current_date: Callable[[], date]`
- `before_iteration(context)` — wraps `_process()` in try/except, logs warning on failure
- `_process(context)` — two-phase logic:
  - Phase 1: auto-decrement all due active buffers with level > 0
  - Phase 2: re-read post-decrement state, collect alertable buffers, inject into prompt

## Key Decisions

1. **No `get_cognitive_state` dependency**: Research correctly identified this as unnecessary. State indicator is already in the prompt from StateResponseHook. SOUL.md handles state-aware presentation. No hook-level state gating needed.

2. **No `get_current_time` dependency**: Buffers use date-based due detection, not time-based. Simplifies the constructor.

3. **Two-phase approach (decrement then alert)**: Phase 1 decrements all due buffers. Phase 2 re-reads from store to get post-decrement state for alert evaluation. This ensures a buffer at level 2 with threshold 1 correctly triggers an alert after decrementing to 1.

4. **Explicit level-0 guard before decrement**: Rather than relying on try/except around `decrement()`, the hook checks `buffer_level > 0` first. This prevents masking unexpected errors and allows continuing to process remaining buffers.

5. **Test file split**: Research warned about exceeding 300 lines (scheduling hook tests were flagged). Split into `test_buffer_hook.py` (pure functions, 191 lines) and `test_buffer_hook_lifecycle.py` (hook + workspace, 272 lines). Both under 300.

6. **Factual alert text**: Alert lines contain only data (name, level/capacity, due date, interval). No guilt phrases, no urgency language. Tests explicitly verify no "urgent", "warning", "danger", "failing", "behind" appear.
