## Files Created

### `schedule_engine.py`
State-aware scheduling decision layer. Contains:
- **`ScheduleAction`** — Pydantic model: `action` (fire/defer/modify/suppress), `reason` (human-readable), `modified_scope` (optional, "reduced" or "single_item")
- **`CheckInContext`** — Pydantic model carrying assembled data per check-in type: pending/in-progress/completed-today/overdue tasks + deadline/energy memories
- **`sort_tasks_by_priority()`** — Sorts by priority tier (high→low), then due-date-present before absent, then earliest due date first
- **`filter_tasks_completed_today()`** — Filters tasks by `updated_at.date() == today`
- **`filter_overdue_tasks()`** — Filters pending tasks with `due_date.date() < today`
- **`evaluate_checkin()`** — 6×4 state×type decision matrix returning ScheduleAction
- **`assemble_checkin_context()`** — Reads from TaskStore/MemoryEntryStore (injected, read-only) to build per-type CheckInContext

## Key Decisions

1. **Flat ScheduleAction model** (research Approach A): Single Pydantic model with `modified_scope: str | None` rather than union types. Matches codebase style and avoids over-engineering for 4 action variants.

2. **`modified_scope` semantic values**: `"reduced"` for avoidance (ICNU framing, shorter scope) and `"single_item"` for overwhelm. These are string conventions the downstream heartbeat hook (sub-03) will match on.

3. **Naive date filtering**: `filter_tasks_completed_today` compares `updated_at.date()` against the caller-provided `today: date`. Timezone conversion is the caller's responsibility, matching the pattern from sub-01 where `is_checkin_due` takes explicit `current_date`.

4. **Avoidance evening_review fires normally**: The matrix fires evening_review during avoidance (not modify), since end-of-day review is inherently limited in scope and provides closure.

5. **`assemble_checkin_context` stores as parameters**: Stores are injected, not globally accessed. The function calls read-only methods only — no mutations.
