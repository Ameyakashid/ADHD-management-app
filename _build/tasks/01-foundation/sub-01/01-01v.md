## Issues Found

### Issue #1 — Contradictory default parameter rule
- **Severity:** MEDIUM
- **File:** `_build/code-rules.md`, lines 12 and 58
- **Description:** Line 12 (Universal Rules) states "Never use default parameter values — make all parameters explicit." Line 58 (Python-Specific Rules) states "No mutable default arguments. Use None sentinel + explicit creation inside the function body." These contradict each other: the first bans ALL defaults, the second explicitly allows `None` as a default. The Python ecosystem uses default parameters pervasively (e.g., `timeout: int = 30`). Banning them entirely is impractical and will cause downstream implementation agents to write awkward code or violate one rule to satisfy the other.
- **Suggested fix:** Change line 12 to "Avoid hidden mutable defaults — use `None` sentinel for mutable arguments, explicit values for immutable ones." This aligns the universal rule with the Python-specific guidance.
