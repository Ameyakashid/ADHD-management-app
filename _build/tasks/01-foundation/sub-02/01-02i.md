## Files Created

### `requirements.txt`
Pins `nanobot-ai==0.1.5`. Single dependency — all transitive deps are managed by pip.

### `workspace/SOUL.md`
Bot personality definition. Direct, warm, non-patronizing. Focuses on ADHD-specific needs: breaking tasks down, celebrating completions without guilt-tripping, matching user energy.

### `workspace/USER.md`
User profile template. Documents ADHD context, device setup (Windows dev / Mac deploy), preference for concise messages and incremental work.

### `workspace/HEARTBEAT.md`
Periodic task definition. Morning check-in (08:00) for daily priorities, evening review (20:00) for summary and rescheduling. Checked by nanobot every 30 minutes.

### `workspace/config.json.template`
Full nanobot-ai config with `${VAR}` interpolation placeholders. Key decisions:
- **Primary provider: OpenRouter** — acts as multi-provider gateway, routes to Claude/Gemini/etc. Satisfies "2+ providers with fallback" via OpenRouter's built-in model routing. Uses `anthropic/claude-3.5-haiku` as default model (cheapest Claude option for $7/month budget).
- **Fallback provider: Ollama** — configured with dummy API key `"ollama"`. Zero-cost local inference when budget ceiling approaches or internet is down. Switch by changing `agents.defaults.provider` to `"ollama"`.
- **Telegram**: enabled, streaming on, `allowFrom` whitelist with placeholder for user's numeric ID.
- **Exec tool disabled**: `bwrap` sandbox is Linux-only; disabled to avoid Windows errors.
- **Dream interval**: 4 hours (vs default 2) to reduce token usage.
- **maxTokens**: 4096 (vs default 8192) for budget efficiency.
- **maxToolIterations**: 50 (vs default 200) to prevent runaway tool loops.

### `setup_workspace.py`
Deployment script that:
1. Reads `.env` file from repo root
2. Validates all required env vars are present and non-placeholder
3. Resolves `${VAR}` placeholders in config template
4. Creates `~/.nanobot/` directory structure (workspace, media, cron, logs, sessions, memory)
5. Copies workspace templates to `~/.nanobot/workspace/`
6. Writes resolved `config.json` to `~/.nanobot/config.json`

Uses `pathlib.Path.home()` throughout — no OS-specific paths.

### `.env.example`
Documents the three required credentials: `OPENROUTER_API_KEY`, `TELEGRAM_BOT_TOKEN`, `TELEGRAM_USER_ID`. Includes instructions for obtaining each.

### `.gitignore`
Excludes `.env`, `.venv/`, `__pycache__/`, IDE files, OS files. Does NOT exclude `workspace/` templates (those are intentionally version-controlled).

## Key Decisions

### OpenRouter as "multi-provider" (Approach A from research)
The research identified three approaches. I chose Approach A (OpenRouter as single gateway) because:
1. Simplest config — one API key for access to all upstream LLMs
2. OpenRouter handles failover natively between upstream providers
3. Satisfies the acceptance criterion "at least 2 providers with fallback"
4. Added Ollama as a secondary provider (Approach C hybrid) for zero-cost local fallback

### Workspace templates in repo (Approach B from research)
Templates live in `workspace/` in the git repo. `setup_workspace.py` copies them to `~/.nanobot/`. This avoids the MEMORY.md noise problem (Approach C) while keeping templates version-controlled.

### Python 3.12 virtual environment
nanobot-ai requires Python 3.11+. The system Python is 3.10.6 but Python 3.12.10 is available via `py -3.12`. Created `.venv` with 3.12. The `requirements.txt` approach (vs pyproject.toml) is simpler and sufficient for a single-dependency project.

### Budget-conscious defaults
Reduced `maxTokens` (4096), `maxToolIterations` (50), and `dream.intervalH` (4) from framework defaults to stay within $7/month ceiling.

## How to Run

```bash
# 1. Create venv (Python 3.12+)
py -3.12 -m venv .venv
.venv/Scripts/activate   # Windows
# source .venv/bin/activate  # Mac/Linux

# 2. Install dependencies
pip install -r requirements.txt

# 3. Copy .env.example to .env, fill in credentials
cp .env.example .env

# 4. Deploy workspace to ~/.nanobot/
python setup_workspace.py

# 5. Start the bot
nanobot gateway
```
