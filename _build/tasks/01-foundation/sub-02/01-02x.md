## Decisions

```
DECISION: OpenRouter as primary provider, Ollama as local fallback
REASON: OpenRouter acts as multi-provider gateway natively (handles failover between upstream LLMs). Ollama provides zero-cost local inference when budget ceiling approaches. Satisfies "2+ providers with fallback" criterion.

DECISION: Workspace templates in repo + setup script deploys to ~/.nanobot/
REASON: Keeps templates version-controlled while avoiding dream system's auto-modification of MEMORY.md polluting git diffs. Approach B from research.

DECISION: Budget-conscious agent defaults (maxTokens=4096, maxToolIterations=50, dream.intervalH=4)
REASON: Reduced from framework defaults (8192, 200, 2) to stay within $7/month budget ceiling. Model is anthropic/claude-3.5-haiku (cheapest Claude option via OpenRouter).

DECISION: Exec tool disabled in config
REASON: bwrap sandbox is Linux-only. Disabled to prevent Windows runtime errors.

DECISION: requirements.txt over pyproject.toml
REASON: Single dependency (nanobot-ai==0.1.5). No build system needed. Simpler.

DECISION: Python 3.12 required (via py -3.12 -m venv)
REASON: nanobot-ai requires Python 3.11+. System Python on dev machine is 3.10.6. Python 3.12.10 available via py launcher.
```
