## Issues Found

### 1. LOW — `.gitignore:14-16`: Dead gitignore patterns
**File:** `.gitignore`, lines 14-16
**Description:** Patterns `~/.nanobot/sessions/`, `~/.nanobot/logs/`, `~/.nanobot/media/` use `~/` prefix which gitignore does not expand. Gitignore patterns are relative to the repo root. Since `~/.nanobot/` lives outside the repo, these lines are harmless noise — they match nothing.
**Fix:** Remove lines 14-16 or replace with a comment noting that `~/.nanobot/` is outside the repo and not tracked.

### 2. LOW — No type checker in dev dependencies
**File:** `requirements.txt`
**Description:** Code rules require `mypy --strict` or `pyright` but neither is installed. The code itself uses correct type annotations (`dict[str, str]`, `list[str]`, `Path`, return types on all functions), so this is an enforcement gap, not a correctness issue.
**Fix:** Add `mypy` or `pyright` to a `requirements-dev.txt` file in a future subtask.

### 3. LOW — Brittle placeholder detection in `validate_env_vars`
**File:** `setup_workspace.py:60`
**Description:** `value == "123456789"` is checked against ALL env vars. While designed to catch users who copied `.env.example` without editing, `123456789` is a valid Telegram user ID range. A user with that exact ID would be blocked from deploying. Extremely unlikely in practice.
**Fix:** Use a more distinctive placeholder like `YOUR_TELEGRAM_USER_ID_HERE` in `.env.example`, or check placeholder substrings rather than exact numeric matches.

### 4. LOW — Naive string replacement in `resolve_config_template`
**File:** `setup_workspace.py:74-78`
**Description:** `raw.replace(placeholder, env[var_name])` does plain text substitution on raw JSON. If an env var value contains a double quote or backslash, the resulting JSON would be malformed. Since input comes from the user's own `.env` file this is self-harm rather than an attack vector, but could produce confusing errors.
**Fix:** Use `json.dumps(value)[1:-1]` to properly escape values before insertion, or build the config dict programmatically instead of doing string replacement.
