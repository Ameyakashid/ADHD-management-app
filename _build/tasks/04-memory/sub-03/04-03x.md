## Index Report: memory-context-injection (04/sub-03)

### What It Is

MemoryContextHook — a nanobot-ai lifecycle hook that reads active structured memories from MemoryEntryStore and injects them into the system prompt on every message. Paired with SOUL.md instructions teaching the bot when/how to use memory tools.

### Key Decisions

1. **Injection point**: After `## Long-term Memory` heading, before next section boundary (`#` or `---`). Falls back to appending if heading absent.
2. **Token budget**: Capped at 20 entries (configurable via constructor). Most recent first.
3. **Error handling**: `before_iteration` wraps `_inject` in try/except, logs warning, continues silently. Bot keeps working even if memory store is corrupted.
4. **No injection for empty**: Zero active entries = no `## Active Memories` section added. Clean prompt.
5. **SOUL.md memory rules**: Explicit instructions for all 5 categories, "confirm with user" mandate, "review periodically" guidance.

### Integration Points

- **Downstream**: Nanobot startup must register `MemoryContextHook` alongside `StateResponseHook`. Both hooks modify `messages[0]` — order matters (state first, then memories, since memories inject after Long-term Memory section while state injects after State-Aware Adaptation section — no conflict).
- **MEMORY.md**: Seeded with User Patterns / Key Facts / Notes sections for Dream's Phase 2 editor.
