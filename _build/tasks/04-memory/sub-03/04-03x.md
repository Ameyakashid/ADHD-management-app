## Section 4: Decisions

```
DECISION: Injection point targets ## Long-term Memory heading, inserts before next section boundary
REASON: nanobot-ai ContextBuilder assembles Identity → Bootstrap (SOUL.md) → Memory (## Long-term Memory + MEMORY.md) → Skills. Active memories belong inside the memory section.

DECISION: Fallback appends at end if ## Long-term Memory heading absent
REASON: MEMORY.md may not exist on fresh installs — graceful degradation over crash.

DECISION: Capped at 20 entries (DEFAULT_MAX_INJECTED_ENTRIES) with most-recent-first ordering
REASON: Token budget — 20 entries × ~25 tokens = ~500 tokens. Single-user bot unlikely to exceed this. Configurable via constructor.

DECISION: HookContext protocol redeclared locally (3 lines) instead of importing from state_response_integration.py
REASON: Avoids coupling two independently-composable hooks. Extract to shared module only when a third hook appears (YAGNI).

DECISION: before_iteration wraps _inject in try/except, logs warning, never crashes
REASON: Matches StateResponseHook error handling pattern. Memory injection failure must not break the bot.

DECISION: format_memory_entries sorts independently even though _inject also sorts before truncating
REASON: format_memory_entries is a public function callable directly — it must guarantee its own sort contract. Double sort on ≤20 items is negligible.

DECISION: No asyncio in pure functions — only before_iteration is async
REASON: format_single_entry, format_memory_entries, inject_memories_into_prompt are synchronous pure functions. Only the hook method is async to match nanobot-ai AgentHook contract.
```
