### Architecture Review

**Hook pattern consistency**: `MemoryContextHook` follows the exact same pattern as `StateResponseHook` — `HookContext` protocol, `before_iteration(context)` async method, `messages[0]` system prompt modification via dict spread. Good.

**Injection placement**: `inject_memories_into_prompt` searches for `## Long-term Memory` heading and inserts the Active Memories block before the next section boundary. Falls back to appending at end. Correct behavior for nanobot's ContextBuilder layout.

**Separation of concerns**: Three pure formatting/injection functions handle data transformation. The `MemoryContextHook` class handles lifecycle (store access, error catching). Clean split.

### Issues

1. **LOW — Double sort**: `_inject` sorts entries by `created_at` descending before truncating (line 117-118), then passes them to `format_memory_entries` which sorts again (line 48). The second sort is redundant. Not a bug — just wasted cycles on a list capped at 20 entries.

2. **LOW — Deprecated asyncio API**: Tests use `asyncio.get_event_loop()` which emits DeprecationWarning on Python 3.10+. Should use `asyncio.run()`. Pre-existing pattern from state_response_integration tests — not introduced by this subtask.

3. **LOW — HookContext protocol duplicated**: Defined identically in `memory_context.py` and `state_response_integration.py`. Acceptable under YAGNI for 2 consumers. Worth extracting to a shared module if a third hook appears.

4. **INFO — `# type: ignore[arg-type]` in test helper**: `_entry()` passes plain string for Literal-typed `category` parameter. Pragmatic for test ergonomics.
