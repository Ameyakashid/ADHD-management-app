## Summary

Three nanobot-ai Tool subclasses wrapping MemoryEntryStore CRUD: SaveMemoryTool (create entry by category+content+metadata), ListMemoriesTool (list active entries with optional category filter, read-only), DismissMemoryTool (resolve entry by ID with graceful KeyError handling). Registered via `register_memory_tools(registry, store)`. 26 tests, all pass.

## Key Decisions

1. **Registry pattern over return-list:** Description spec said `register_memory_tools(store)` returns a list. Implementation uses `register_memory_tools(registry, store)` matching task_tools.py and the actual nanobot-ai ToolRegistry API. Correct deviation.
2. **read_only property:** Only ListMemoriesTool declares `read_only = True`. Save and Dismiss are mutations, correctly omit it (defaults to False in base).
3. **Error as string, not exception:** DismissMemoryTool catches KeyError and returns an error string with guidance. This matches the nanobot-ai convention where `execute()` returns strings and the registry appends "[Analyze the error above...]" hints.
4. **CATEGORY_ENUM as module constant:** Shared between all three tool schemas to stay DRY. Mirrors the MemoryCategory Literal in memory_store.py.

## Integration Notes

- `register_memory_tools()` is not yet called at startup — same deferred wiring as `register_task_tools()`. A future integration subtask will wire both into the nanobot agent loop.
- Tests use real MemoryEntryStore with tmp_path (no mocks), same pattern as task_tools tests.
