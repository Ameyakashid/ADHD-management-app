## Decisions

```
DECISION: Used register_memory_tools(registry, store) instead of returning a list
REASON: Matches established register_task_tools() pattern and nanobot-ai ToolRegistry API. Description spec was outdated.

DECISION: ListMemoriesTool.read_only = True, others omit it
REASON: Only list is a pure read. Save and Dismiss mutate state. Matches nanobot-ai convention where read_only defaults to False.

DECISION: DismissMemoryTool catches KeyError, returns error string instead of raising
REASON: nanobot-ai convention — execute() returns strings, never raises. Error message suggests list_memories for actionable recovery.

DECISION: CATEGORY_ENUM module-level constant shared across all 3 tool schemas
REASON: DRY — avoids repeating the 5-category list in 3 separate schema decorators. Mirrors MemoryCategory Literal in memory_store.py.

DECISION: type: ignore[arg-type] on category parameter in execute() methods
REASON: category arrives as str from LLM but store expects MemoryCategory Literal. Schema enum constraint validates at runtime. Same pattern as task_tools.py.

DECISION: metadata parameter uses ObjectSchema with additional_properties={"type": "string"}
REASON: Allows arbitrary string key-value pairs from LLM. No fixed metadata schema — flexibility for due_date, task_id, source, etc.
```
