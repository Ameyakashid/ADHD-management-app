## Tools Implemented

### SaveMemoryTool (`save_memory`)
- Parameters: `category` (required, enum of 5 types), `content` (required), `metadata` (optional dict)
- Delegates to `store.create_entry()`, returns formatted confirmation with entry ID
- `metadata` parameter uses `ObjectSchema(additional_properties={"type": "string"}, nullable=True)`

### ListMemoriesTool (`list_memories`)
- Parameters: `category` (optional, enum of 5 types)
- `read_only = True`
- Without filter: calls `store.list_active_entries()` (hides resolved)
- With filter: calls `store.list_entries_by_category(category)`
- Returns "No active memories." when empty

### DismissMemoryTool (`dismiss_memory`)
- Parameters: `entry_id` (required)
- Catches `KeyError` from store, returns error string suggesting `list_memories`
- Returns formatted confirmation showing resolved timestamp

## Formatting Helpers

- `format_memory_entry(entry)` — `[id[:8]] category: content` with Created/Resolved/Metadata lines
- `format_memory_list(entries)` — joins entries or returns "No active memories."

## Registration

`register_memory_tools(registry, store)` registers all 3 tools into a `ToolRegistry`. Same pattern as `register_task_tools()`.

## Pattern Adherence

Followed `task_tools.py` exactly: `@tool_parameters` decorator, `Tool` subclass with `name`/`description`/`execute`, `type: ignore[arg-type]` for Literal narrowing, error strings instead of exceptions, `ToolRegistry` registration.
