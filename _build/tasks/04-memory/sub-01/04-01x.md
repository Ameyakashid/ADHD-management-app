## Decisions

```
DECISION: Pydantic + JSON flat file, not SQLite
REASON: Mirrors task_store.py pattern exactly, zero new deps, sufficient for single-user <1000 entries

DECISION: No UpdateModel / partial update — entries are append-only
REASON: Memory entries don't get edited; only mutation is resolve (single field flip)

DECISION: No delete method — soft-delete via resolve_entry only
REASON: Preserves history; resolved entries remain queryable via list_entries()

DECISION: list_entries_by_category returns active entries only
REASON: Primary use case is "show active blockers/deadlines"; list_entries() exists for everything

DECISION: resolve_entry is idempotent (re-resolve updates timestamp)
REASON: Simpler than raising on already-resolved; correct for dismiss/clear semantics

DECISION: metadata typed as dict[str,str] not dict[str,Any]
REASON: Keeps JSON simple, avoids mixed types; numeric values string-encoded by convention

DECISION: Atomic writes via tmp+rename in _save()
REASON: Prevents corrupt JSON on crash; same pattern as task_store.py
```
