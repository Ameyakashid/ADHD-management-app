## Issues Found

### LOW: Mixed types in resolve_entry_model dict (memory_store.py:69-71)
`model_dump(mode="json")` serializes datetimes as ISO strings, then `data["resolved_at"]` is set to a raw `datetime` object. The dict has string datetimes for `created_at` but a datetime object for `resolved_at`. Works because Pydantic coerces both in `model_validate`, but the intermediate dict is type-inconsistent. Same pattern exists in `task_store.py:apply_updates` (line 95-98) — inherited, not introduced.

**Suggested fix**: Use `model_dump()` (Python mode) instead of `model_dump(mode="json")` so all values remain Python objects.

### LOW: Test catches generic Exception (test_memory_model.py:71)
```python
with pytest.raises(Exception):
```
Should be `pytest.raises(pydantic.ValidationError)` for precision. Current test would pass even if constructor threw for an unrelated reason.

### LOW: ALL_CATEGORIES frozenset manually duplicates Literal values (memory_store.py:23-25)
The frozenset and `MemoryCategory` Literal can drift if one is updated without the other. Mitigated by `test_all_categories` which validates contents. Same pattern as `task_store.py`. Consistent, not ideal.

No CRITICAL, HIGH, or MEDIUM issues found.
