## New Issues Found

### 1. MEDIUM — `normalize_llm_response` substring fallback shadows "hyperfocus" with "focus"

- **File:** `state_detection.py:200-202`
- **Description:** The fallback loop iterates `sorted(ALL_STATES)` → `["avoidance", "baseline", "focus", "hyperfocus", ...]` and uses `if state in cleaned` for substring matching. Since `"focus"` is a substring of `"hyperfocus"`, any noisy LLM response containing `"hyperfocus"` (e.g., `"I think hyperfocus"`, `"definitely hyperfocus"`) hits `"focus"` first and returns `"focus"` instead of `"hyperfocus"`.
- **Impact:** Low probability (LLM is instructed to return only the state name, so the exact-match path at line 197-198 handles clean responses), but when triggered it silently returns the wrong state. Markov enforcement may or may not catch it depending on the current state's transition matrix.
- **Suggested fix:** Sort state names by length descending (longest first) so `"hyperfocus"` is checked before `"focus"` in the substring loop. One-line change: `for state in sorted(ALL_STATES, key=len, reverse=True):`.
- **No test exists for this case.** A test like `normalize_llm_response("definitely hyperfocus") == "hyperfocus"` would currently fail.

### 2. LOW — `02-02x.md` exports section lists old `str` signatures

- **File:** `_build/tasks/02-personality/sub-02/02-02x.md:37-40`
- **Description:** The index report still shows `build_classification_prompt(message: str, current_state: str, ...)`, `normalize_llm_response(...) -> str`, etc. These should now show `StateName`. Not a code issue — stale documentation that the index report agent should update.
