## Files Created

### `workspace/states.yaml`
YAML configuration defining all 6 cognitive states: Baseline, Focus, Hyperfocus, Avoidance, Overwhelm, RSD. Each state has:
- `description` — human-readable summary for the LLM prompt
- `detection_signals` — list of message cues the LLM looks for
- `response_style` — how the bot should modify its behavior in this state
- `transitions` — 6x6 Markov transition probability matrix (each row sums to 1.0)

Transition design decisions:
- Self-transitions are 0.30–0.50 to prevent state flickering
- Avoidance, Overwhelm, RSD cannot reach Hyperfocus (0.0 probability) — you can't deep-focus from crisis
- Hyperfocus → Overwhelm exists (0.15) for post-hyperfocus crash
- Avoidance ↔ Overwhelm has moderate bidirectional probability (they feed each other)
- RSD is reachable from any state but with low probability from positive states

### `state_detection.py` (187 lines)
Pure-function module with pydantic models. Components:
- **`CognitiveState`** — pydantic model for one state, validates transitions sum to 1.0, all target states exist, probabilities in [0, 1]
- **`StateConfig`** — pydantic model for the full config, validates all 6 states present
- **`DetectionResult`** — output model with `previous_state`, `detected_state`, `is_transition_blocked`
- **`load_state_config(config_path)`** — loads and validates YAML with `yaml.safe_load` + pydantic
- **`build_classification_prompt(message, current_state, config)`** — constructs the LLM classification prompt including all state descriptions and detection signals
- **`normalize_llm_response(raw_response)`** — extracts valid state name from noisy LLM output (strips whitespace, lowercases, searches for state names in longer responses)
- **`enforce_transition(current_state, llm_detected, config)`** — blocks transitions with 0.0 probability, returns current state instead
- **`detect_state(message, current_state, config, llm_call)`** — async orchestrator, takes an `LLMCallable` protocol for provider decoupling
- **`LLMCallable`** — Protocol type for the async LLM call dependency

### `tests/test_state_detection.py` (92 tests)

## Key Decisions

1. **YAML over JSON** — followed research recommendation. States need inline documentation (comments) and human-editable descriptions. YAML supports both.

2. **Single-shot classification prompt (Approach A)** — followed research recommendation. Simpler, cheaper, sufficient. The Markov matrix provides the "prior" constraint; confidence scores from Approach B aren't needed because impossible transitions are hard-blocked at 0.0 probability.

3. **LLMCallable Protocol** — `detect_state` takes an async callable `(str) -> str` instead of importing nanobot's provider directly. This keeps the module testable without nanobot installed and decouples from provider details.

4. **`StateName` as `Literal` type** — used `Literal["baseline", "focus", ...]` for type safety, plus `ALL_STATES: frozenset` for runtime lookups.

5. **No `Any` types** — `StateConfig.states` uses `dict[str, CognitiveState]` not `dict[str, Any]`. `LLMCallable` is a Protocol, not a bare callable.

6. **`asyncio.get_event_loop().run_until_complete()`** in tests — used instead of pytest-asyncio markers because the existing test suite uses synchronous pytest. The 7 deprecation warnings are harmless and can be addressed later by adding `@pytest.mark.asyncio`.
