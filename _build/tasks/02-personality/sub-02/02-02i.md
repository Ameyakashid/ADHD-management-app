## Changes Made

### 1. Split test file to comply with 300-line limit (MEDIUM → FIXED)

`tests/test_state_detection.py` was 439 lines. Split at the natural boundary (line ~232):

- **`tests/test_state_config.py`** (224 lines) — TestConfigFileStructure (21 tests), TestTransitionMatrix (28 tests), TestConfigValidation (5 tests). Config loading, YAML structure, and Markov matrix validation.
- **`tests/test_state_detection.py`** (235 lines) — TestPromptConstruction (5 tests), TestNormalizeLlmResponse (20 tests), TestEnforceTransition (4 tests), TestDetectState (7 tests). Detection logic, LLM normalization, transition enforcement.

Both files are under the 300-line limit.

### 2. Protocol import reorganized (LOW → FIXED)

`from typing import Protocol` was at line 240, separated from `from typing import Literal` at line 12. Merged into a single import: `from typing import Literal, Protocol` at line 12. `LLMCallable` class moved from end-of-file (lines 243-246) to after `DetectionResult` and before `load_state_config`, eliminating the forward reference string annotation in `detect_state`.

### 3. DetectionResult typed with StateName (LOW → FIXED)

`previous_state` and `detected_state` fields changed from `str` to `StateName` (the `Literal` type). Provides static type safety — pydantic validates at construction that only valid state names are used.

### 4. Deprecated asyncio.get_event_loop() replaced (LOW → FIXED)

`TestDetectState._run_async()` used `asyncio.get_event_loop().run_until_complete()`, deprecated since Python 3.10. Replaced with `asyncio.run()` in both `tests/test_state_detection.py` and `tests/test_state_response_integration.py` (which had the same pattern and was failing when run after the other test module due to event loop closure).

### 5. Non-deterministic normalize_llm_response fixed (LOW → FIXED)

The fallback loop in `normalize_llm_response` iterated `ALL_STATES` (a `frozenset` — unordered). If LLM output contained multiple state names, the returned state depended on iteration order. Changed to `sorted(ALL_STATES)` for deterministic alphabetical-first matching.
