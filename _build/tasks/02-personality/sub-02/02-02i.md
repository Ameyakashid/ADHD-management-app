## Changes Made

### 1. Function parameters tightened from `str` to `StateName` (LOW -> FIXED)

Three functions had `current_state: str` where `StateName` (a `Literal` type) would catch invalid state names at static analysis time:

- `build_classification_prompt` (line 126): `current_state: str` -> `current_state: StateName`
- `enforce_transition` (line 157): `current_state: str` -> `current_state: StateName`, `llm_detected: str` -> `llm_detected: StateName`
- `detect_state` (line 212): `current_state: str` -> `current_state: StateName`

### 2. `normalize_llm_response` return type tightened (NEW)

Return type changed from `str` to `StateName`. The function always returns a valid state name (either from the direct `ALL_STATES` membership check or from the `sorted(ALL_STATES)` fallback loop). The tighter return type documents this contract and lets static checkers propagate the narrowing to callers.
