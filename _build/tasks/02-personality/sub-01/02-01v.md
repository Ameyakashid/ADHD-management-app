## Issues Found

### LOW: Incomplete banned-phrase coverage in USER.md tests
- **File:** `tests/test_personality.py:168-176`
- **Description:** `TestNoBannedLanguageInUserMd.BANNED_PHRASES` checks 6 of the 10 banned phrases from SOUL.md. Missing: "just do it", "why didn't you", "why can't you", "you forgot again", "I already told you", "all you have to do".
- **Impact:** None currently — USER.md does not contain any of these phrases (verified via grep). This is a coverage gap, not a functional bug.
- **Suggested fix:** Add the missing 4 phrases to `BANNED_PHRASES` for regression protection.

### LOW: Alternatives section test checks only 3 of 10 banned phrases
- **File:** `tests/test_personality.py:84-88`
- **Description:** `TestSoulNoBannedPatternsInAlternatives.banned_as_instruction` checks "you should", "just do it", "try harder" but not the other 7 banned phrases. If someone later edits the alternatives section to include "why can't you" as an instruction, it would go undetected.
- **Impact:** None currently — the alternatives section is clean. Coverage gap only.
- **Suggested fix:** Extend the banned pattern list or loop over the same `BANNED_ROOTS` from `TestSoulBannedPatterns`.
