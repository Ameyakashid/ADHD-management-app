## Files Created

### start.py (68 lines)
Cross-platform Python launcher that runs nanobot gateway (subprocess) and dashboard server (thread) in parallel. Dashboard uses `create_dashboard_server()` directly via threading — no extra process. Signal handler (SIGINT/SIGTERM) cleanly shuts down both. If nanobot exits, dashboard thread is joined and process exits with nanobot's code.

## Key Decisions

1. **Config endpoint over template injection**: Research recommended Option A (server-side config endpoint). Implemented as a direct handler in `do_GET` rather than adding to `dispatch_route`, because `/config` takes no `data_dir` parameter — different signature from data endpoints. This avoided changing `dispatch_route` and its existing tests.

2. **Default parameter on make_handler_class**: Added `refresh_interval_ms=30000` default to `make_handler_class` so existing tests that call it with 2 args continue to work without modification.

3. **Thread-based dashboard in launcher**: Used `threading.Thread` + `handle_request()` loop with 1s timeout rather than `serve_forever()`. This allows clean shutdown via `threading.Event` without needing `server.shutdown()` from the signal handler (which can deadlock on some platforms).

4. **Dead code cleanup**: Removed `ROUTES` dict and `RouteHandler` alias (flagged as dead code in sub-01 verification). This was necessary to stay under 300 lines after adding the config endpoint. Also removed 9 unused imports.
