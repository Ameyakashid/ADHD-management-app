## Decisions

```
DECISION: /config endpoint instead of template injection for refresh interval
REASON: Keeps static files static. One extra HTTP request on page load vs breaking clean static serving pattern. Frontend falls back to 30000ms if /config fails.

DECISION: Config endpoint handled inline in do_GET before dispatch_route
REASON: /config takes no data_dir parameter — different signature from data endpoints. Avoids changing dispatch_route and its existing tests.

DECISION: Thread-based dashboard + subprocess nanobot in start.py
REASON: dashboard_api already exposes create_dashboard_server() — import directly, no extra process. handle_request() loop with 1s timeout + threading.Event for clean shutdown without serve_forever() deadlock risk.

DECISION: Default parameter refresh_interval_ms=30000 on make_handler_class
REASON: Backward compatibility — 26 existing tests call make_handler_class with 2 args. code-rules.md contradicts itself on defaults (line 12 vs 58). Immutable int, documented.

DECISION: Removed ROUTES dict and 9 unused imports from dashboard_api.py
REASON: Dead code flagged in sub-01 verification. Necessary to stay under 300 lines after adding /config endpoint. Net -10 lines.

DECISION: Added "data" dir to setup_workspace.py create_required_dirs()
REASON: Dashboard API reads from data/ dir. Without this, first API request could 404 if no store has written yet. Stores create lazily but dashboard reads eagerly.
```
