## Files Created

### task_store.py (210 lines)

Task data model and persistent JSON storage. Contains:

- **Types:** `TaskStatus = Literal["pending", "in_progress", "done"]`, `TaskPriority = Literal["low", "medium", "high"]`, plus `ALL_STATUSES`/`ALL_PRIORITIES` frozensets
- **Task model:** Pydantic BaseModel with id, title, description, status, priority, created_at, updated_at, due_date, tags
- **TaskUpdate model:** Pydantic model for partial updates — uses `exclude_unset=True` on dump to distinguish "not provided" from "set to None"
- **Pure functions:** `generate_task_id()`, `build_task()`, `apply_updates()`, `serialize_tasks()`, `deserialize_tasks()` — all testable without disk I/O
- **TaskStore class:** CRUD operations (create_task, get_task, list_tasks, list_tasks_by_status, update_task, mark_complete, delete_task) with write-through JSON persistence and atomic writes

## Key Decisions

1. **Pydantic over stdlib dataclass.** Research recommended this for consistency with state_detection.py patterns. The description says "dataclass" but code-rules allow either, and every existing model uses Pydantic.

2. **Literal types over StrEnum.** Follows the established `StateName = Literal[...]` pattern from state_detection.py.

3. **TaskUpdate model with exclude_unset.** Solves the "set to None vs. don't change" ambiguity cleanly using Pydantic's built-in `model_dump(exclude_unset=True)`. No sentinel values needed.

4. **Pure functions extracted from TaskStore.** `build_task`, `apply_updates`, `serialize_tasks`, `deserialize_tasks` are all pure and testable without disk. TaskStore is thin — it delegates to these and manages I/O.

5. **All create_task parameters explicit, no defaults.** Follows the "never use default parameter values" rule. Caller passes `None` and `[]` explicitly for optional fields.

6. **Atomic writes.** `_save` writes to `.tmp` then uses `Path.replace()` (which wraps `os.replace`, atomic on NTFS and POSIX).

7. **Test file split.** Initial test file was 352 lines. Split into test_task_model.py (pure functions/models, 193 lines) and test_task_store.py (CRUD/persistence, 185 lines) to stay under 300-line limit.
