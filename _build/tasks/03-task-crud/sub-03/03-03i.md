## Files Modified

### workspace/SOUL.md
Added `## Task Management` section between Boundaries and State-Aware Adaptation (Option A from research). The section contains four subsections:

1. **When to Offer Task Creation** — offer to capture when user mentions something to do, confirm first, one at a time
2. **Presenting Task Lists** — max 5 tasks per message, lead with most relevant, short format, summarize overflow
3. **Handling Completions** — brief acknowledgment, no over-celebration, mention natural next step once at most
4. **State-Aware Task Behavior** — per-state rules for all 6 cognitive states:
   - Baseline: normal task actions
   - Focus: only active tasks, no new introductions
   - Hyperfocus: no task mentions unless hard deadline
   - Avoidance: use ICNU, offer smallest first step, no full lists
   - Overwhelm: show at most one task, no choices
   - RSD: no tasks/deadlines/incomplete work until user re-engages

SOUL.md grew from 114 to ~140 lines, well under the 300-line limit. No banned phrases in the new content (verified by 20 parametrized tests). Existing section headings unchanged — all personality tests pass.

## Key Decisions

1. **Placement: Option A** — Task Management before State-Aware Adaptation. General task rules in one place; state-specific task behavior within the section references the state system without duplicating the general state rules already in State-Aware Adaptation.

2. **Nanobot skip strategy** — The current environment has Python 3.10 and nanobot-ai requires 3.11+. Used `try/except ImportError` with `pytest.mark.skipif` so SOUL.md content tests always run while tool integration tests skip gracefully. The 3 nanobot-dependent tests will run on any environment with nanobot-ai installed.

3. **No test duplication** — Existing test_task_tools.py has 34 tests covering individual tool behavior and registration. The integration tests focus exclusively on the cross-tool cycle and persistence — the things not yet tested.

4. **Banned phrase list expanded** — The research identified 11 banned roots from test_personality.py. The integration test uses the full 20-phrase list from SOUL.md (including "just try", "you always forget", "you need to try", etc.) for thorough coverage since the existing personality tests only scan the "Say This Instead" subsection, not new content sections.
